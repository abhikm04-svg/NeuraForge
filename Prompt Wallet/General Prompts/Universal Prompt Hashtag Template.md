**1. Context Setting**

- '# Background → Relevant context, history, or why this task matters.
- '# Objective → The specific goal of this prompt.

**2. Instructions to the Model**

- '# Instructions → The core “do this” section.
- '# Style → Tone, voice, or format preferences (e.g., concise, executive-ready, explanatory).
- '# Output Format → Expected structure (bullets, table, narrative, JSON, etc.).

**3. Domain Guidance**

- '# Skills and Areas of Knowledge → What expertise you want the model to lean on (e.g., product strategy, SQL optimization, pedagogy).
- '# Assumptions → Any explicit assumptions to use when information is missing.

**4. Evaluation and Guardrails**

- '# Constraints → What NOT to do (e.g., don’t invent numbers, keep under 500 words).
- '# Metrics of Success → How to know if the output is good (e.g., clarity, alignment to KPIs).

**5. Optional Add-ons**

- '# Examples → Any sample inputs/outputs to anchor responses.
- '# Audience → Who the final output is for (execs, students, engineers).
- '# Next Steps → What the user might do with the output (review, refine, present).
